# Learning ledger

## Candidate learning

- Stable candidate ID:
- Date, plugin version, and affected skill:
- Detection source and mistake class:
- Protected foundation IDs affected:
- Coarse context and project-only scope:
- Expected versus observed result:
- Redacted evidence class and project-local `lin-` ID:
- Result versus interpretation:
- Root cause:
- Strongest counterexplanation tested:
- Proposed rule and applicability:
- Exceptions, confidence, and reversal condition:
- Regression test ID and result:
- Privacy review and redactions:
- Promotion status, decision, and owner:
- Review/expiry date:

Do not store raw private transcripts, credentials, secrets, or customer lists.
Store redacted, practical signals only. A candidate is not an instruction until
it clears the evidence gate and is promoted. Local learning stays project-
scoped; public promotion requires a separate sanitized preview and maintainer
review.

Reuse one random `lin-` ID for every observation from the same experiment,
artifact, cohort, or report. Use a new ID only for a genuinely independent
origin. Never derive the ID from private or customer data.
